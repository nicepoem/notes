# 初始化仓库

对开发者而言，将本地工程同步至（GitHub、Gitee）等远程代码仓库，是一项必备的基础素养。这既能实现代码的云端容灾，又能为版本迭代留下清晰的“履历”，更是多人协作开发的高效枢纽。

本文将详细介绍如何本地项目完整地上传到 GitHub 上的一个新的空白仓库。

默认已经安装 git，并配置好用户信息（用户名和邮箱），后续用来展示提交记录信息所有者。

```
git config --global user.name "username"
git config --global user.email "mail@qq.com"
```

## 初始化本地 Git 仓库

首先进入本地项目的根目录下，然后打开命令行终端（`Git Bash`、`CMD`、`PowerShell`）

执行初始化命令

```bash
git init
```

执行完毕后会在你的项目目录下创建一个隐藏的 `.git` 文件夹，标志着你的项目现在是一个 Git 仓库了。

## 添加所有文件到暂存区

```bash
git add .
```

## 提交更改

 将暂存区中的文件提交到本地仓库的历史记录中，每次提交都代表项目的一个版本。

```bash
git commit -m "Initial commit"
```
- commit 是提交命令。

- `-m` 参数后面跟着的是本次提交的说明信息。请将 `Initial commit` 替换为你本次提交的简短描述，例如重构了代码。

## 关联本地仓库与 GitHub 上的仓库

在 GitHub 创建完空白仓库后显示的页面上，你会找到一串指令，告诉你如何将一个已有的本地仓库推送到这个远程仓库。找到类似下面这样的命令：

```bash
git remote add origin https://github.com/GitHub用户名/仓库名.git
```

- git remote add 命令用于添加一个远程仓库。
- origin 是给这个远程仓库起的一个别名。

你可以通过 git remote -v 命令来查看你当前仓库关联的远程仓库信息：

```bash
git remote -v
```

## 将本地代码推送到 GitHub 仓库

现在，你的本地仓库已经知道远程仓库在哪里了（通过 origin 别名），接下来就是将本地的代码推送到远程仓库。

通常，新创建的 GitHub 仓库默认的主分支是 main。你的本地仓库在 git init 后，默认的主分支通常是 master 或 main（新版 Git 默认 main）。

首先，确认你的本地主分支名称：

```bash
git branch
```

情况一：本地分支是 main，远程默认分支也是 main (推荐，新版Git默认)

```bash
git push -u origin main
```
情况二：本地分支是 master，远程默认分支是 main (旧版Git常见)
你需要指定将本地的 master 分支推送到远程的 main 分支：

```bash
git push -u origin master:main
```
或者，你可以先将本地的 master 分支重命名为 main，然后再执行推送到 main 的命令：

```
git branch -M main # 将当前本地分支重命名为 main
git push -u origin main # 推送到远程的 main 分支
```
